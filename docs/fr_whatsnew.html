- **Native JSON Support**: Added full keyword highlighting and autocompletion for native SQL-standard JSON functions (`JSON_VALUE`, `JSON_QUERY`, `JSON_OBJECT`, `JSON_ARRAY`, `JSON_EXISTS`, `JSON_TABLE`, etc.). This support is available for both Firebird 5.0 and 6.0 databases.
- **SQL:2023 Compliance**: Many of these new features follow the SQL:2023 standard, and FlameRobin's SQL editor has been updated to reflect these latest improvements in the Firebird engine.
- **Improved Tokenizer Robustness**: The SQL tokenizer has been enhanced to correctly handle multi-character operators like `=>`, `||`, `:=`, `>=` and `<=`, even when written without surrounding spaces. This ensures more accurate statement parsing and highlighting.

Firebird 5.0 SQL Editor Enhancements

- **LATERAL JOIN Support**: The SQL editor tokenizer and parser have been updated to correctly handle the new `LATERAL` join syntax introduced in Firebird 5.0. This ensures accurate statement splitting and avoids false "incomplete statement" warnings during script execution.
- **SKIP LOCKED Highlighting**: Added syntax highlighting for the `SKIP LOCKED` clause in `SELECT ... WITH LOCK`, `UPDATE`, and `DELETE` statements. This improves readability for high-concurrency SQL queries.
- **Multiple-Row DML RETURNING**: Added support for displaying multi-row results from `INSERT`, `UPDATE`, and `DELETE` statements that use a `RETURNING` clause. These results are now correctly displayed in the SQL results grid, matching the behavior for standard `SELECT` statements.
- **SQL/PSQL Profiler**: Integrated the Firebird 5.0 built-in profiler (`RDB$PROFILER` package). Users can now enable profiling for their SQL sessions and view detailed line-by-line PSQL execution statistics and record source performance data in a dedicated "Profiler" tab.
- **Version-Aware Keywords**: The keyword set is now dynamically adjusted based on the ODS version of the active connection, preventing keyword conflicts when working with older Firebird versions.

Firebird 5.0 Parallel Operations Support

- **Parallel Workers UI**: Exposed the *parallel worker count* option in the Backup, Restore, and the new Maintenance dialogs. This leverages the `ParallelWorkers` service parameter introduced in Firebird 5.0 to speed up operations on multi-core systems.
- **New Maintenance Dialog**: Introduced a dedicated "Maintenance..." option in the Database tools menu, allowing manual Sweep and Validation/Repair tasks with support for parallel processing.
- **Inline ODS Upgrade**: Exposed the new Firebird 5.0 "upgrade ODS" option in the Maintenance dialog. This allows users to upgrade a database to the latest On-Disk Structure version without a full backup and restore cycle.
- **IBPP Enhancements**: Updated the underlying IBPP library to correctly handle the new Firebird 5.0 SPB parameters for parallel backup and repair tasks.

- Firebird 5 partial index support: generated index DDL now preserves the WHERE predicate so recreated schemas no longer lose partial-index conditions (issue #392)

Details of the partial index fix:

Firebird 5 partial indexes were not fully extracted: generated index DDL omitted the index predicate, so recreated schemas lost WHERE conditions.
This change adds Firebird 5-aware metadata loading and emits partial-index predicates in extracted CREATE INDEX statements.

Metadata loading (Firebird 5 / ODS 13.1+)
- Extend index metadata queries to read RDB$INDICES.RDB$CONDITION_SOURCE
- Gate the column by ODS version (>= 13.1) and fall back to NULL on older servers to keep compatibility
- Store condition text in Index (conditionM) with accessor support

DDL generation
- Centralize index body rendering in a helper (buildIndexBodySql)
- Emit: regular index columns or COMPUTED BY ... ; optional WHERE <condition> when a partial predicate exists
- Apply the same rendering path across index extraction call sites to keep output consistent

Create statement template update
- Update index create-statement template text to include optional WHERE search_condition

Regression coverage
- Add a focused test target (index_ddl_test) covering: regular index body, partial index body, computed + partial index body

Example of generated output:
CREATE INDEX IDX_T_A ON T (A,B) WHERE A IS NOT NULL;
CREATE INDEX IDX_T_EXPR ON T COMPUTED BY UPPER(A) WHERE A IS NOT NULL;
