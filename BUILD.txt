===================================
== FlameRobin Build Instructions ==
===================================

Below are build instructions for FlameRobin on all supported build environments.

--------------------------
-- Common to all builds --
--------------------------

-- General Requirements

Required:
 * FirebirdSQL, the client libs (e.g. fbclient.dll) at a minimum, but the whole
   server install to test with - https://www.firebirdsql.org/
 * A C++14 compiler (Visual Studio on Windows, G++ on Linux, others may work but untested)
   https://visualstudio.microsoft.com/ https://gcc.gnu.org/
 * The wxWidgets library - https://www.wxwidgets.org/
 * CMake - https://cmake.org/
 * git - https://git-scm.com/
Optional:
 * GitHub desktop app (for interacting with this project) - https://github.com/
 * TortoiseGit (Windows extension) - https://tortoisegit.org/

-- Notes

FlameRobin is currently compiled as C++14 (`CMAKE_CXX_STANDARD=14`).
Please keep code changes compatible with C++14.

FR must be built with the same settings for debug/release etc.  as the
wxWidgets library (Unicode build is required). Failing to do so will usually
lead to compile or link errors. This does not apply completely to Linux/GCC
build, as you can build wx in release mode and FlameRobin in debug mode - in
case you just want to debug FR code, and not code of wxWidgets.

If you have problems building FR, please contact the development team:
https://github.com/mariuz/flamerobin

-----------------------------------------
-- MSW - Visual C++ Build Instructions --
-----------------------------------------

FlameRobin now uses:
 * CMake
 * C++20
 * vcpkg manifest mode for dependencies
 * a custom overlay port for the patched fb-cpp library

The current Windows build no longer follows the older manual wxWidgets setup as the
primary path. The recommended configuration is via CMake + vcpkg.

-- Requirements

Required:
 * Visual Studio with Desktop C++ support
 * CMake
 * Git
 * vcpkg
 * A Firebird installation for runtime/testing

Notes:
 * FlameRobin now builds as C++20.
 * The build expects the repository root to contain:
   - vcpkg.json
   - vcpkg-configuration.json
   - ports/fb-cpp
 * The fb-cpp dependency is provided through the repo's vcpkg overlay port.

-- Initial setup

1. Clone FlameRobin:
   > git clone https://github.com/mariuz/flamerobin.git
   > cd flamerobin

2. Obtain vcpkg if you do not already have it:
   > git clone https://github.com/microsoft/vcpkg C:\src\vcpkg
   > C:\src\vcpkg\bootstrap-vcpkg.bat

-- Configure with CMake

Use a clean out-of-source build directory:

   > cmake -S . -B build -G "Visual Studio 18 2026" -A x64 -DCMAKE_TOOLCHAIN_FILE=C:\src\vcpkg\scripts\buildsystems\vcpkg.cmake

This enables vcpkg manifest mode so CMake can install and find dependencies,
including the patched fb-cpp package from the repository's overlay port.

-- Build

   > cmake --build build --config Release

-- Troubleshooting

If CMake fails with an error like:

   Could not find a package configuration file provided by "fb-cpp"

then the vcpkg toolchain was not active during configure.

Check the configure output for:
   DEBUG: CMAKE_TOOLCHAIN_FILE=...
   DEBUG: VCPKG_TARGET_TRIPLET=...
   DEBUG: VCPKG_INSTALLED_DIR=...

If those values are empty, delete the build directory and re-run CMake with
-DCMAKE_TOOLCHAIN_FILE pointing to your vcpkg checkout.

Always reconfigure from a clean build directory after changing toolchain settings.
---------------------------------------------------
-- Unix (Debian and related; Ubuntu, Mint, etc.) --
---------------------------------------------------

Assumes a 'project' dir in you home directory, change to suit

 1. Setup the tools and libs needed:
    $ sudo apt-get install build-essential git cmake libwxgtk3.2-dev
    (Firebird client and other dependencies will be handled by vcpkg)
 2. $ cd ~/project
 3. $ git clone --recursive https://github.com/mariuz/flamerobin.git
 4. $ cd flamerobin
 5. $ mkdir build
 6. $ cd build
 7. $ cmake ..
    Note: vcpkg is enabled by default. If you need to disable it, use -DENABLE_VCPKG=OFF,
    but you will need to provide all dependencies (including fb-cpp) manually.
 8. $ make
 9. To run FlameRobin use the run_flamerobin.sh script. This sets up an
    environment variable that points the FlameRobin app to the build directory
    which contains various sub-dir used by the app
    $ ./run_flamerobin.sh

-- Install support

The usual 'make install' will work as normal, you may want to configure the
build for 'release':
 $ cmake -DCMAKE_BUILD_TYPE=Release ..
 $ make
 $ sudo make install

------------
-- Mac OS --
------------

Assumes Homebrew (https://brew.sh/) is installed.
Tested on Apple Silicon (arm64) with macOS 26.3.1.

-- Firebird client library

FlameRobin requires the Firebird client library (libfbclient) at build
and runtime.

    System Firebird install (https://www.firebirdsql.org/):
    The installer places the framework at:
    /Library/Frameworks/Firebird.framework/
    CMake will find it automatically.


-- Build steps

 1. Install the required tools and libraries:
    $ brew install cmake wxwidgets
 2. $ cd ~/projects
 3. $ git clone https://github.com/mariuz/flamerobin.git
 4. $ cd flamerobin
 5. $ mkdir build
 6. $ cd build
 7. $ cmake -DCMAKE_PREFIX_PATH="$(brew --prefix)" ..
    CMake will locate libfbclient automatically from the Firebird
    framework or Homebrew. If your Firebird client is in a custom
    location, set CMAKE_PREFIX_PATH or FBLIB accordingly:
    $ cmake -DCMAKE_PREFIX_PATH="/path/to/firebird" ..
 8. $ make
 9. To run FlameRobin directly from the build directory:
    $ open flamerobin.app

-- Install to /Applications (optional)

    $ sudo cmake --install .

-- Notes

 * The app bundle embeds an rpath pointing to where libfbclient was
   found at build time, so the app runs without DYLD_LIBRARY_PATH.
 * If you move or reinstall Firebird after building, rebuild from step 7.
